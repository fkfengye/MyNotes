Java内存模型与硬件存储关联关系

JMM、JVM内存结构和硬件存储的多层次映射解析

Java内存模型 (JMM)

主内存 (Main Memory)

JVM对应： 堆内存、方法区（存储共享变量）

硬件对应： 主内存(RAM) + CPU缓存(L3)

特性： 所有线程共享，volatile变量直接存储在主内存

工作内存 (Working Memory)

JVM对应： 虚拟机栈的部分区域（局部变量表）

硬件对应： CPU寄存器 + 高速缓存(L1/L2)

特性： 线程私有，存储主内存数据的副本

内存屏障 (Memory Barriers)

JVM实现： LoadLoad, StoreStore, LoadStore, StoreLoad屏障

硬件实现： CPU指令 (mfence, lfence, sfence)

作用： 控制指令重排序和内存可见性

JVM内存结构

堆 (Heap)

JMM对应： 主内存（存储对象实例）

硬件对应： 主内存(RAM) + CPU缓存(L3)

特性： GC管理，所有线程共享

虚拟机栈 (JVM Stack)

JMM对应： 工作内存（局部变量）

硬件对应： CPU寄存器 + 高速缓存(L1)

特性： 线程私有，存储栈帧

方法区 (Method Area)

JMM对应： 主内存（存储类信息、常量）

硬件对应： 主内存(RAM) + 磁盘（虚拟内存）

特性： JDK8后由元空间实现
