依赖关系说明

这张图展示了从硬件层到Java应用层的并发控制依赖关系：

  • 自下而上依赖：上层组件依赖于下层提供的机制
  • 硬件层（MESI协议）：CPU缓存一致性协议，是Java内存模型的基础
  • happens-before原则：JVM内存模型的核心理论，建立在硬件内存模型之上
  • JVM内存屏障：基于happens-before原则实现，用于控制指令执行顺序
  • Java应用底层机制：基于JVM内存屏障实现，包括synchronized、CAS和volatile
  • Java应用中层的原子类和AQS：基于底层的CAS和volatile实现
  • Java应用上层的ReentrantLock：基于AQS实现
  • 线程同步工具：基于AQS和ReentrantLock等机制实现
